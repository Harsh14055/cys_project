🎯 DETECTION FLOW
═══════════════════════════════════════════════════════════════════

User uploads file
       ↓
   [Signature Check]
   (SHA256 hash lookup)
       ├─ MATCH FOUND → Report known malware/benign
       │
       └─ NO MATCH → Continue to ML analysis
       ↓
   [ML Analysis]
   ├─ Extract 20 features
   ├─ Scale features using StandardScaler
   ├─ Feed to Random Forest model
   └─ Get prediction + confidence
       ↓
   [Confidence-Based Verdict]
   ├─ Confidence > 70% → High confidence verdict
   ├─ 40-70% → Suspicious (review recommended)
   └─ < 40% → Likely benign
       ↓
   [Final Verdict]
   ├─ MALICIOUS (known or high ML confidence)
   ├─ BENIGN (known safe or low malware probability)
   ├─ SUSPICIOUS (uncertain, manual review needed)
   └─ UNKNOWN (new file, ML unavailable)

⚙️ TECHNICAL DETAILS
═══════════════════════════════════════════════════════════════════

Feature Vector (20 dimensions):
  1. file_size (bytes)
  2. file_size_kb (KB)
  3. entropy (0-8)
  4. magic_bytes (categorical: 0-9)
  5. section_count (PE sections)
  6. suspicious_strings_count
  7. null_byte_ratio (0-1)
  8. printable_ratio (0-1)
  9. has_mz_header (binary)
  10. has_elf_header (binary)
  11. has_jar_header (binary)
  12. is_small_file (binary)
  13. is_medium_file (binary)
  14. is_large_file (binary)
  15-19. top_byte_0..4_freq (5 features)
  20. avg_byte_value (0-255)

Classification:
  - Output: Binary [0=Benign, 1=Malware]
  - Confidence: Probability score (0-1)
  - Final Verdict: Combined with signature results
